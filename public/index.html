About this prototype

This is the decision layer: given a proposed action plus pre-shaped context, it returns a routing verdict (execute silently, execute and notify, confirm, ask clarifying, or refuse) with a concise rationale.

The context pipeline (conversation history + user state → populated task schema) is deliberately stubbed. Preloaded scenarios hand in pre-shaped inputs; the custom form lets you build a conversation by hand. In production, entity_id would come from retrieval; policies would be composable; user trust would be learned per-user. See the README for the full scope writeup.

Design principle: we control the schema, the LLM populates it, we control what to do with it.
